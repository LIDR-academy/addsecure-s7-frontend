# Position Board Page

## Route
`dashboard/positions/:id`

---

## Interview Flow (Kanban Columns)

**Endpoint:** `GET /positions/:id/interviewFlow`

- Fetch interview steps and render them as **Kanban columns**
- Display the **position name** as the page headline
- Include a **Go Back** button that navigates to `/positions`

**Response shape:**
```json
{
  "positionName": "Senior backend engineer",
  "interviewFlow": {
    "id": 1,
    "description": "Standard development interview process",
    "interviewSteps": [
      { "id": 1, "interviewFlowId": 1, "interviewTypeId": 1, "name": "Initial Screening", "orderIndex": 1 },
      { "id": 2, "interviewFlowId": 1, "interviewTypeId": 2, "name": "Technical Interview", "orderIndex": 2 },
      { "id": 3, "interviewFlowId": 1, "interviewTypeId": 3, "name": "Manager Interview", "orderIndex": 2 }
    ]
  }
}
```

---

## Candidates (Cards)

**Endpoint:** `GET /positions/:id/candidates`

- Fetch candidates and **organize them into the matching Kanban column** based on `currentInterviewStep`
- Each card displays:
  - Candidate **full name**
  - **Average score**
- Use the provided Figma image as design inspiration

**Response shape:**
```json
[
  { "fullName": "Jane Smith",    "currentInterviewStep": "Technical Interview", "averageScore": 4 },
  { "fullName": "Carlos García", "currentInterviewStep": "Initial Screening",   "averageScore": 0 },
  { "fullName": "John Doe",      "currentInterviewStep": "Manager Interview",   "averageScore": 5 }
]
```

---

## Drag & Drop — Stage Update

**Endpoint:** `PUT /candidates/:id/stage`

- Enable **drag and drop** between columns
- On drop, call the endpoint with:

```json
{
  "applicationId": "1",
  "currentInterviewStep": "3"
}
```

- **Response:**
```json
{
  "message": "Candidate stage updated successfully",
  "data": {
    "id": 1,
    "positionId": 1,
    "candidateId": 1,
    "applicationDate": "2024-06-04T13:34:58.304Z",
    "currentInterviewStep": 3,
    "notes": null,
    "interviews": []
  }
}
```

- On success, **display a toast notification** with the response message
